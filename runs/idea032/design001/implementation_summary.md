**Changes:**
- `code/pelvis_utils.py`: Added `import torch.nn.functional as F` and appended a `downsample_depth_map(depth_map, feat_h, feat_w)` helper that bilinearly downsamples a `(B, 1, H, W)` depth map to `(B, feat_h, feat_w)` for use as the per-feature-cell reconstruction target.
- `code/pose3d_transformer_head.py`: Added `import torch.nn.functional as F`, extended the `pelvis_utils` import to also bring in `downsample_depth_map`, registered a module-global `forward_pre_hook` (guarded by `_RGBD_CAPTURE_HOOK_REGISTERED`) that captures any 4-channel 4-D float tensor into `_LAST_RGBD_INPUT[device]` so the head can access the raw RGBD input's depth channel without modifying invariant estimator/preprocessor/transform files. Extended `Pose3dTransformerHead.__init__` with nine aux-depth kwargs (`use_aux_depth`, `aux_depth_loss_weight`, `aux_depth_log_space`, `aux_depth_grad_weight`, `aux_depth_valid_min`, `aux_depth_valid_max`, `aux_depth_denorm_scale`, `feat_h`, `feat_w`), stored them on `self`, and instantiated a zero-initialised `self.aux_depth_head = nn.Linear(hidden_dim, 1)` when enabled. In `forward()`, after `spatial = spatial + pos_enc`, computed `self._aux_depth_pred = self.aux_depth_head(spatial).squeeze(-1).view(B, feat_h, feat_w)` when enabled, else None. In `loss()`, after the UV-loss line and before the `with torch.no_grad()` MPJPE block, read the captured RGBD tensor, extracted channel 3, multiplied by `aux_depth_denorm_scale=20.0` to recover metres, bilinearly downsampled, masked to `(0.1, 30.0)`, applied `SmoothL1(beta=0.1)` (with empty-mask `pred.sum()*0.0` safety), optionally added a log-space target and/or a gradient-consistency term, and wrote `losses['loss/aux_depth/train'] = aux_depth_loss_weight * recon_loss`. `predict()` path untouched.
- `code/config.py`: Appended `use_aux_depth=True, aux_depth_loss_weight=0.1, aux_depth_log_space=False, aux_depth_grad_weight=0.0, aux_depth_valid_min=0.1, aux_depth_valid_max=30.0, aux_depth_denorm_scale=20.0, feat_h=40, feat_w=24` to the `head=dict(...)` block. All values are literals; no new imports.
