- `code/pose3d_transformer_head.py` — Identical head implementation as design001/design002 (the file is shared across all three designs of idea013 and behaviour is selected by config flags). For this design the `per_limb_heads=True` branch is active: `__init__` asserts `kinematic_parametrization=True`, validates `limb_index` (len 22, values in `[0, num_limbs)`, `num_limbs == 5`, covers `{0..21}` exactly), allocates `self.body_limb_heads = nn.ModuleList` of 5 `Linear(hidden_dim, 3)` heads, registers `self.limb_index` as a non-persistent long buffer, pre-computes `self._limb_token_lists` (Python `list[list[int]]`), and registers five non-persistent per-limb index buffers `_limb_idx_0.._limb_idx_4` (one long tensor per limb) to avoid per-forward allocation. `_init_head_weights` trunc-normal-inits each per-limb head with `std=0.02` then applies the `1/sqrt(21)` in-place scaling (under `torch.no_grad()`) alongside the shared `joints_out.weight`. In `forward()`, under `per_limb_heads=True`, body tokens are routed per-limb: iterate the five limb token lists, `index_select` the relevant decoder features, push through `body_limb_heads[limb_id]`, and scatter the `(B, k, 3)` outputs into a `(B, 22, 3)` canvas via the out-of-place `Tensor.index_copy` (autograd-safe, the result is reassigned each loop). Hand tokens still go through the shared `self.joints_out`. The body canvas is passed through `_forward_kinematics` and concatenated with the hand coords along `dim=1`. `bone_length_loss_weight=0.0` so the bone-length auxiliary loss is off, and `predict()` is unchanged.
