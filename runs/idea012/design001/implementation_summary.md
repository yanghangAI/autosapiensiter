**Changes:**

- `code/pose3d_transformer_head.py`: Added three new keyword-only kwargs to `Pose3dTransformerHead.__init__` (`dist_loss_weight: float = 0.0`, `dist_loss_mode: str = 'abs'`, `dist_loss_eps: float = 1e-3`), stored them as attributes, and added an `assert` that `dist_loss_mode in ('abs', 'bone_weighted', 'log')`. Inside `loss()`, after the existing three loss-dict assignments and before the `with torch.no_grad():` MPJPE block, appended a `if self.dist_loss_weight > 0.0:` guarded block that computes the pairwise Euclidean distance matrix with `torch.cdist(pred_body, pred_body, p=2)` and `torch.cdist(gt_body, gt_body, p=2)` on the 22 body joints, gathers the 231 upper-triangular entries via `torch.triu_indices(22, 22, offset=1, device=pred_body.device)`, and — under mode `'abs'` — stores `losses['loss/dist_matrix/train'] = self.dist_loss_weight * (d_pred - d_gt).abs().mean()`. The full three-branch `if/elif/else` on `self.dist_loss_mode` (`'abs'`, `'bone_weighted'`, `'log'`) is present so the head file skeleton is shared across Designs 001/002/003; only the `'abs'` branch is exercised in Design 001 since that is the configured mode. `forward()`, `predict()`, `_init_head_weights`, and the rest of the file are unchanged. No new learnable parameters or buffers are introduced in Design 001.
- `code/config.py`: Added three int/str/float literal kwargs at the end of the `head=dict(...)` block inside `model=dict(...)` — `dist_loss_weight=0.5`, `dist_loss_mode='abs'`, `dist_loss_eps=1e-3` — to enable the new auxiliary pairwise distance-matrix loss with the Design 001 scalar weight. All other config values (optimizer, LR schedule, data pipeline, hooks, dataloaders, evaluators, seed, batch size, accumulation, custom_imports) are byte-identical to the baseline.
